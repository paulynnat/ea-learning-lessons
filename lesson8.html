What is GitHub?

GitHub is a website that hosts Git repositories in the cloud. It lets you:

  • Back up your code online
  • Share your projects (e.g., in a portfolio or with a team)
  • Collaborate with others using the same Git workflow
  • Use GitHub Pages to host simple websites for free

Git is the tool (version control). GitHub is the place where you store and share your Git repositories.

Create a GitHub Account

  1. Go to github.com
  2. Click Sign up and create an account (email, password, username)
  3. Verify your email if prompted

Your profile URL will be https://github.com/your-username.

Create a New Repository

A repository (repo) is one project on GitHub.

  1. On GitHub, click the + in the top-right and choose New repository
  2. Give it a name (e.g. my-first-website)
  3. Add a short description if you like
  4. Choose Public
  5. Do not check "Add a README" or "Add .gitignore" if you already have a local project—you'll push existing code
  6. Click Create repository

GitHub will show you a page with commands. You'll use the "push an existing repository" section.

Connect Your Local Repo to GitHub (Remote)

Your code lives in two places: local (your computer) and remote (GitHub). You link them once, then push and pull.

1. Add the Remote

In your project folder (where you ran git init), run this—replace YOUR-USERNAME and YOUR-REPO-NAME with your GitHub username and repo name:

git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git

Example: If your username is johndev and repo is my-first-website:

git remote add origin https://github.com/johndev/my-first-website.git

origin is the default name for the main remote. You only run this once per project.

2. Verify the Remote

git remote -v

You should see origin and the GitHub URL.

Push Your Code to GitHub

Push sends your commits from your computer to GitHub.

First Push (Set Up the Branch)

If this is the first time pushing, you may need to set the default branch name and push:

# Make sure you have at least one commit
git add .
git commit -m "Initial commit"

# Push to GitHub (first time)
git branch -M main
git push -u origin main

-u origin main means: use origin and the branch main, and remember this so next time you can just run git push.

Later Pushes

After you make more commits locally:

git add .
git commit -m "Add new lesson page"
git push

Refresh your repo on GitHub—you'll see your latest files and commits.

Authentication: Logging In to GitHub from the Terminal

When you run git push, Git needs to prove you're allowed to change that repo. GitHub no longer accepts account passwords for this.

Option 1: HTTPS with Personal Access Token (PAT)

  1. On GitHub: Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Click Generate new token (classic)
  3. Give it a name (e.g. "My laptop"), choose an expiry, and check repo
  4. Generate and copy the token (you won't see it again!)
  5. When you git push, use your GitHub username and paste the token as the password

You can save the token in your system's credential manager so you don't type it every time.

Option 2: SSH Key (Recommended for Regular Use)

SSH lets you push without typing a password each time.

  1. Generate a key: ssh-keygen -t ed25519 -C "your.email@example.com" (press Enter to accept defaults)
  2. Start the agent and add the key: eval "$(ssh-agent -s)" then ssh-add ~/.ssh/id_ed25519
  3. Copy your public key: cat ~/.ssh/id_ed25519.pub (copy the whole line)
  4. On GitHub: Settings → SSH and GPG keys → New SSH key → paste and save
  5. Change your remote to SSH: git remote set-url origin git@github.com:YOUR-USERNAME/YOUR-REPO-NAME.git

After that, git push uses your SSH key instead of a password.

Beginner tip: Start with a Personal Access Token (HTTPS). You can switch to SSH later when you're comfortable.

Clone: Get a Copy of a Repo on Your Computer

If a project already exists on GitHub and you want to work on it locally:

# Clone via HTTPS
git clone https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git

# Clone via SSH (if you use SSH keys)
git clone git@github.com:YOUR-USERNAME/YOUR-REPO-NAME.git

This creates a folder with the repo name and downloads all files and history. Then cd YOUR-REPO-NAME and start working. When you make changes, use git add, git commit, and git push as before.

Pull: Get Updates from GitHub

If you (or someone else) made changes on GitHub and you want those changes on your computer:

git pull origin main

Or, if you already set main as upstream: git pull.

Full Workflow: Local Server → Commit → Push

Here's the complete flow from "run locally" to "code on GitHub":

# 1. Run your project locally (e.g. Live Server, or python3 -m http.server 8000)
# 2. Make changes and test in the browser

# 3. When happy, save a snapshot with Git
git add .
git status                    # optional: check what you're committing
git commit -m "Describe your change"

# 4. Send to GitHub
git push

Repeat steps 2–4 whenever you finish a chunk of work. Your code is versioned locally and backed up on GitHub.

Quick Reference

git remote add origin https://github.com/USER/REPO.git   # Link local to GitHub (once)
git remote -v                                              # Show remotes
git branch -M main                                         # Rename branch to main (once)
git push -u origin main                                    # First push
git push                                                   # Later pushes
git pull                                                   # Get updates from GitHub
git clone https://github.com/USER/REPO.git                 # Get a copy of a repo

Exercise 1: Push This Project to GitHub

Your Task:

  1. Create a new repository on GitHub (e.g. ea-learning-lessons)
  2. In this project folder, run git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO.git
  3. Run git branch -M main then git push -u origin main
  4. Log in with your username and Personal Access Token if asked
  5. Refresh the repo on GitHub and confirm all lesson files appear

Exercise 2: Change Something and Push Again

Your Task:

  1. Add a single line to README.md (e.g. "I completed the GitHub lessons.")
  2. Run git add README.md, then git commit -m "Update README", then git push
  3. On GitHub, open README.md and confirm your change is there

Exercise 3: Clone on Another Folder (Optional)

Your Task:

Simulate getting the project on a "new computer":

  1. Create a new folder somewhere (e.g. Desktop/my-copy)
  2. Open terminal in that folder and run git clone https://github.com/YOUR-USERNAME/YOUR-REPO.git
  3. Enter the cloned folder and open index.html—you should see the same project

What's Next?

You're Done With the GitHub Basics!

You can now run your project on a local server and push it to GitHub. Keep practicing:

  • Make small commits with clear messages
  • Push regularly so your work is backed up
  • Try GitHub Pages to host a static site from a repo (Settings → Pages → source: main branch)
  • Explore branches and pull requests when you're ready to collaborate
