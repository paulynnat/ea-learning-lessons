Why Run a Local Server?

So far you've been opening HTML files directly in the browser (e.g., file:///Users/you/project/index.html). That works, but:

  • Some features (like fetching data or certain JavaScript APIs) require a proper HTTP URL
  • Relative paths and links behave more like they will on a real website
  • You can test your site the way visitors will see it

Ways to Run a Local Server

Option 1: VS Code Live Server (Easiest)

If you use Visual Studio Code (or Cursor):

  1. Install the Live Server extension (by Ritwick Dey)
  2. Right-click your index.html (or any HTML file)
  3. Select "Open with Live Server"
  4. Your browser opens at something like http://127.0.0.1:5500 and auto-refreshes when you save files
Tip: You can also click "Go Live" in the bottom status bar of VS Code/Cursor to start the server.

Option 2: Python (Built-in on Mac/Linux)

If you have Python installed, open a terminal in your project folder and run:

# Python 3
python3 -m http.server 8000

Then open http://localhost:8000 in your browser. Use Ctrl+C in the terminal to stop the server.

Option 3: Node.js (npx serve)

If you have Node.js installed:

npx serve

Follow the URL shown in the terminal (e.g., http://localhost:3000).

Summary: Pick one method that works on your computer. For beginners, Live Server in VS Code/Cursor is usually the simplest.

What is Git?

Git is a version control system. It helps you:

  • Save snapshots of your project (called commits) so you can go back if something breaks
  • See what changed between versions
  • Collaborate with others without overwriting each other's work
  • Back up and share your code (e.g., on GitHub)

Think of it like "Save" in a game—but you keep every save and can jump back to any of them.

Install Git

  • Mac: Install Xcode Command Line Tools: run xcode-select --install in Terminal, or install from git-scm.com
  • Windows: Download and run the installer from git-scm.com
  • Linux: sudo apt install git (Ubuntu/Debian) or your package manager

Check that Git is installed:

git --version

You should see something like git version 2.x.x.

Configure Git (First Time Only)

Tell Git your name and email so your commits are labeled correctly:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Use the same email you'll use for GitHub.

Basic Git Workflow

You'll use these steps over and over:

  1. Initialize a repo in your project (once)
  2. Add files you want to save
  3. Commit to create a snapshot

1. Initialize a Repository

Open a terminal in your project folder (the folder that contains index.html) and run:

git init

This creates a hidden .git folder. Your project is now a Git repository.

2. Check Status

See which files are new or changed:

git status

New and modified files appear as "untracked" or "changes not staged for commit".

3. Add Files

Stage the files you want to include in your next snapshot:

# Add a single file
git add index.html

# Add all files in the current folder
git add .

# Add all HTML files
git add *.html

After git add, run git status again—you'll see files listed as "staged for commit".

4. Commit

Create a snapshot with a short message describing what you did:

git commit -m "Add first lesson and index page"

Every commit needs a message. Use present tense and be clear: "Add contact form", "Fix button style", "Update Lesson 3".

Quick Reference

git init              # Turn current folder into a Git repo (once)
git status             # See what's changed
git add .              # Stage all changes
git add filename       # Stage one file
git commit -m "msg"    # Save a snapshot with a message

.gitignore

A .gitignore file tells Git which files not to track (e.g., system files, secrets, build output).

Create a file named .gitignore in your project root with one line per pattern:

# OS files
.DS_Store
Thumbs.db

# Editor folders
.vscode/
.idea/

# Logs and temp files
*.log
node_modules/

Then run git add . and git commit -m "Add .gitignore". Git will ignore the listed files.

Exercise 2: Make Your First Commit

Your Task:

In a copy of this project (or a new folder with a simple index.html):

  1. Open terminal in that folder and run git init
  2. Run git status and note which files are listed
  3. Create a .gitignore with at least .DS_Store (Mac) or Thumbs.db (Windows)
  4. Run git add . then git commit -m "Initial commit"
  5. Run git status again—it should say "nothing to commit, working tree clean"
